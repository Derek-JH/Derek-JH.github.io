本网站并未完全按照老师所发的参考文档配置。 发布方式为GitHub pages。

# 为什么要搭建一个个人的博客

   作为一个立志参加IT行业的准程序员，拥有一个属于自己的博客是一件十分不错的事情，不但能够提升自己的讲述能力，而且还可以记录下自己的操作方便日后查看。同时还是有：

  1. 可以放上简历，让自己的建立更好看。
  2. 记录更多的技术，便于自己日后研究。
  3. 分享自己所收获的经验，与其他人进行讨论提升自我。

# 何为VuePress？我们为什么要使用VuePress？

   VuePress是由尤雨溪所发布的基于Vue框架(vue.js百度百科)驱动的静态网站生成器，内置webpack可以用于编写文档，参考文档 (VuePress中文参考文档)。
  我们为什么要使用VuePress进行平台的搭建呢，主要是因为我们所必须要用到的虚拟主机可以省略，而且使用起来非常的简洁、方便，作为新手上手速度快，在多端查看方便。

   首先使用到的是NPM，npm是随同NodeJS一起安装的包管理工具，可以解决NodeJS代码部署上的很多问题(npm)。

vuepressBlogDemo
├─── docs
│ ├── README.md

│  ├──images

   在config.js文件中配置网站的标题、主体信息

module.exports = {
    title: 'Derek\'s blog',
    description: '我的个人网站',
    head: [ // 注入到当前页面的 HTML <head> 中的标签
		['link', { rel: 'icon', href: '/logo.jpg' }], // 增加一个自定义的 favicon(网页标签的图标)
		['link',{rel:''}],
	],
	serviceWorker: true,
    base: '/', // 这是部署到github相关的配置
    markdown: {
      lineNumbers: true // 代码块显示行号
    },
    themeConfig: {
      nav:[ // 导航栏配置
        {text: '博客', link: '/blog_heart/' },
        {text: 'SLOGAN', link: '/SLOGAN/'},
        {text: '微博', link: 'https://baidu.com'}      
      ],
      sidebar: {
		  '/SLOGAN/':[
			'',
			'one'
		  ],
		  '/blog_heart/':[
			'',
			'one',
			'中国芯'
		  ]
	  }, // 侧边栏配置
      sidebarDepth: 2, // 侧边栏显示2级
    }
  };

# 部署上线

注意

本人选择发布在Github pages上，其他发布方法请参考VuePress部署

   在Github上建立新的仓库，仓库名为USERNAME.github.io

   使用Github上传到刚才建立的仓库中，然后npm run dev运行即可。
